13 /*
14  * This API allows you to get regular callbacks during the flight loop, the
15  * part of X-Plane where the plane's position calculates the physics of
16  * flight, etc. Use these APIs to accomplish periodic tasks like logging data
17  * and performing I/O.
18  *
19  * You can receive a callback either just before or just after the per-frame
20  * physics calculations happen - you can use post-FM callbacks to "patch" the
21  * flight model after it has run.
22  *
23  * If the user has set the number of flight model iterations per frame greater
24  * than one your plugin will _not_ see this; these integrations run on the
25  * sub-section of the flight model where iterations improve responsiveness
26  * (e.g. physical integration, not simple systems tracking) and are thus
27  * opaque to plugins.
28  *
29  * Flight loop scheduling, when scheduled by time, is scheduled by a "first
30  * callback after the deadline" schedule, e.g. your callbacks will always be
31  * slightly late to ensure that we don't run faster than your deadline.
32  *
33  * WARNING: Do NOT use these callbacks to draw! You cannot draw during flight
34  * loop callbacks. Use the drawing callbacks (see XPLMDisplay for more info)
35  * for graphics. (One exception: you can use a post-flight loop callback to
36  * update your own off-screen FBOs.)
37  *
38  */

82 /*
83  * XPLMFlightLoop_f
84  *
85  * This is your flight loop callback. Each time the flight loop is iterated
86  * through, you receive this call at the end.
87  *
88  * Flight loop callbacks receive a number of input timing parameters. These
89  * input timing parameters are not particularly useful; you may need to track
90  * your own timing data (e.g. by reading datarefs). The input parameters are:
91  *
92  * - inElapsedSinceLastCall: the wall time since your last callback.
93  * - inElapsedTimeSinceLastFlightLoop: the wall time since any flight loop was
94  * dispatched.
95  * - inCounter: a monotonically increasing counter, bumped once per flight
96  * loop dispatch from the sim.
97  * - inRefcon: your own ptr constant from when you regitered yor callback.
98  *
99  * Your return value controls when you will next be called.
100  *
101  * - Return 0 to stop receiving callbacks.
102  * - Pass a positive number to specify how many seconds until the next
103  * callback. (You will be called at or after this time, not before.)
104  * - Pass a negative number to specify how many loops must go by until you
105  * are called. For example, -1.0 means call me the very next loop.
106  *
107  * Try to run your flight loop as infrequently as is practical, and suspend it
108  * (using return value 0) when you do not need it; lots of flight loop
109  * callbacks that do nothing lowers X-Plane's frame rate.
110  *
111  * Your callback will NOT be unregistered if you return 0; it will merely be
112  * inactive.
113  *
114  */
115 typedef float (* XPLMFlightLoop_f)(
116  float inElapsedSinceLastCall,
117  float inElapsedTimeSinceLastFlightLoop,
118  int inCounter,
119  void * inRefcon);
