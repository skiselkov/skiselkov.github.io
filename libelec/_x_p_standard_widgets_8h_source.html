13 /*
14  * ## THEORY OF OPERATION
15  *
16  * The standard widgets are widgets built into the widgets library. While you
17  * can gain access to the widget function that drives them, you generally use
18  * them by calling XPCreateWidget and then listening for special messages,
19  * etc.
20  *
21  * The standard widgets often send mesages to themselves when the user
22  * performs an event; these messages are sent up the widget hierarchy until
23  * they are handled. So you can add a widget proc directly to a push button
24  * (for example) to intercept the message when it is clicked, or you can put
25  * one widget proc on a window for all of the push buttons in the window. Most
26  * of these messages contain the original widget ID as a parameter so you can
27  * know which widget is messaging no matter who it is sent to.
28  *
29  */

140 /*
141  * The button class provides a number of different button styles and
142  * behaviors, including push buttons, radio buttons, check boxes, etc. The
143  * button label appears on or next to the button depending on the button's
144  * appearance, or type.
145  *
146  * The button's behavior is a separate property that dictates who it hilights
147  * and what kinds of messages it sends. Since behavior and type are different,
148  * you can do strange things like make check boxes that act as push buttons or
149  * push buttons with radio button behavior.
150  *
151  * In X-Plane 6 there were no check box graphics. The result is the following
152  * behavior: in X-Plane
153  * 6 all check box and radio buttons are round (radio-button style) buttons;
154  * in X-Plane 7 they are all square (check-box style) buttons. In a future
155  * version of X-Plane, the xpButtonBehavior enums will provide the correct
156  * graphic (check box or radio button) giving the expected result.
157  *
158  */

266  * The text field widget provides an editable text field including mouse

268  * descriptor. (The descriptor changes as the user types.)
269  *
270  * The text field can have a number of types, that effect the visual layout of
