13 /*
14  * A few notes on coordinate systems:
15  *
16  * X-Plane uses three kinds of coordinates. Global coordinates are specified
17  * as latitude, longitude and elevation. This coordinate system never changes
18  * but is not very precise.
19  *
20  * OpenGL (or 'local') coordinates are cartesian and shift with the plane.
21  * They offer more precision and are used for 3-d OpenGL drawing. The X axis
22  * is aligned east-west with positive X meaning east. The Y axis is aligned
23  * straight up and down at the point 0,0,0 (but since the earth is round it is
24  * not truly straight up and down at other points). The Z axis is aligned
25  * north-south at 0, 0, 0 with positive Z pointing south (but since the earth
26  * is round it isn't exactly north-south as you move east or west of 0, 0, 0).
27  * One unit is one meter and the point 0,0,0 is on the surface of the earth at
28  * sea level for some latitude and longitude picked by the sim such that the
29  * user's aircraft is reasonably nearby.
30  *
31  * 2-d Panel coordinates are 2d, with the X axis horizontal and the Y axis
32  * vertical. The point 0,0 is the bottom left and 1024,768 is the upper
33  * right of the screen. This is true no matter what resolution the user's
34  * monitor is in; when running in higher resolution, graphics will be
35  * scaled.
36  *
37  * Use X-Plane's routines to convert between global and local coordinates. Do
38  * not attempt to do this conversion yourself; the precise 'roundness' of
39  * X-Plane's physics model may not match your own, and (to make things
40  * weirder) the user can potentially customize the physics of the current
41  * planet.
42  *
43  */

89 /*
90  * XPLMSetGraphicsState
91  *
92  * XPLMSetGraphicsState changes OpenGL's fixed function pipeline state. You
93  * are not responsible for restoring any state that is accessed via
94  * XPLMSetGraphicsState, but you are responsible for not accessing this state
95  * directly.
96  *
97  * - inEnableFog - enables or disables fog, equivalent to: glEnable(GL_FOG);
98  * - inNumberTexUnits - enables or disables a number of multitexturing units.
99  * If the number is 0, 2d texturing is disabled entirely, as in
100  * glDisable(GL_TEXTURE_2D); Otherwise, 2d texturing is enabled, and a
101  * number of multitexturing units are enabled sequentially, starting with
102  * unit 0, e.g. glActiveTextureARB(GL_TEXTURE0_ARB); glEnable
103  * (GL_TEXTURE_2D);
104  * - inEnableLighting - enables or disables OpenGL lighting, e.g.
105  * glEnable(GL_LIGHTING); glEnable(GL_LIGHT0);
106  * - inEnableAlphaTesting - enables or disables the alpha test per pixel, e.g.
107  * glEnable(GL_ALPHA_TEST);
108  * - inEnableAlphaBlending - enables or disables alpha blending per pixel,
109  * e.g. glEnable(GL_BLEND);
110  * - inEnableDepthTesting - enables per pixel depth testing, as in
111  * glEnable(GL_DEPTH_TEST);
112  * - inEnableDepthWriting - enables writing back of depth information to the
113  * depth bufffer, as in glDepthMask(GL_TRUE);
114  *
115  * The purpose of this function is to change OpenGL state while keeping
116  * X-Plane aware of the state changes; this keeps X-Plane from getting
117  * surprised by OGL state changes, and prevents X-Plane and plug-ins from
118  * having to set all state before all draws; XPLMSetGraphicsState internally
119  * skips calls to change state that is already properly enabled.
120  *
121  * X-Plane does not have a 'default' OGL state for plug-ins with respect to
122  * the above state vector; plug-ins should totally set OGL state using this
123  * API before drawing. Use XPLMSetGraphicsState instead of any of the above
124  * OpenGL calls.
125  *
126  * WARNING: Any routine that performs drawing (e.g. XPLMDrawString or widget
127  * code) may change X-Plane's state. Always set state before drawing after
128  * unknown code has executed.
129  *
130  * *Deprecation Warnings*: X-Plane's lighting and fog environemnt is
131  * significantly more complex than the fixed function pipeline can express;
132  * do not assume that lighting and fog state is a good approximation for 3-d
133  * drawing. Prefer to use XPLMInstancing to draw objects. All calls to
134  * XPLMSetGraphicsState should have no fog or lighting.
135  *
136  */
137 XPLM_API void XPLMSetGraphicsState(
138  int inEnableFog,
139  int inNumberTexUnits,
140  int inEnableLighting,
141  int inEnableAlphaTesting,
142  int inEnableAlphaBlending,
143  int inEnableDepthTesting,
144  int inEnableDepthWriting);
