35 /*
36  * OpenGL VERSIONS
37  *
38  * OpenGL comes in two major flavors:
39  *
40  * 1) Modern OpenGL (3.2+)
41  * 2) Legacy OpenGL (2.0 and earlier)
42  *
43  * There are major incompatibilities between the two branches and
44  * you SHOULD always strive to write modern OpenGL code, not legacy.
45  * On Windows and Linux, you can request a modern OpenGL context
46  * (version 4.0 or later) and most reasonably recent drivers will
47  * transparently support legacy OpenGL constructs (unless you set
48  * fwd_compat=B_TRUE, in which case we will instruct the driver to
49  * explicitly reject legacy features).
50  *
51  * However, on MacOS this separation is enforced with an iron hand.
52  * So on MacOS, glctx_create_invisible ignores the version arguments
53  * except for major_ver:
54  * 1) if you specify major_ver <= 2, you will get OpenGL legacy
55  * (version 1.0 up to 2.1 core features)
56  * 2) if you specify major_ver = 3, you will get OpenGL 3.2+
57  * feature compatibility
58  * 3) in all other cases, you will get up to OpenGL 4.1
59  * feature compatibility
60  * Please note that MacOS doesn't support any later OpenGL revisions,
61  * so the maximum you can get reliably is the OpenGL 4.1 core profile.
62  */
63 
64 /*
65  * Creates an invisible OpenGL context of a specified version in a
66  * platform-independent way. Please note that this must be called on
67  * the main X-Plane thread. To create a background rendering thread,
68  * call glctx_create_invisible() on the main thread, then hand off the
69  * created glctx_t to the background thread, which will subsequently
70  * set it as its current context using glctx_make_current().
71  *
72  * @param win_ptr An opaque window pointer to a backing window that the
73  * OpenGL will be based on. The context will NOT be drawing to this
74  * window. The window handle is only used to fetch hardware context
75  * information such as the target GPU and supported capabilities.
76  * Unless you know exactly what you are doing, you should always
77  * pass the return value of glctx_get_xplane_win_ptr() here.
78  * @param share_ctx An optional OpenGL context that you want the new
79  * context to share resources with. Please note that this will also
80  * share the command pipeline between the contexts, which can
81  * introduced undesirable command contention, so use with care.
82  * @param major_ver Major OpenGL version requested. See "OpenGL VERSIONS".
83  * @param minor_ver Major OpenGL version requested. See "OpenGL VERSIONS".
84  * @param fwd_compat If set to true, the returned OpenGL context will NOT
85  * be backwards compatible. See "OpenGL VERSIONS" for more info.
86  * @param debug Enables additional error checking and reporting on
87  * platforms that support it (Linux & Windows), often at the cost of
88  * some performance. Enable on debug code, disable on production code.
89  *
90  * @return On success, an initialized OpenGL context. On failure, the
91  * function returns NULL and prints an error reason to the log.
92  * The returned context will NOT yet have been made the current
93  * context. To make the context current, use glctx_make_current.
94  */
95 API_EXPORT glctx_t *glctx_create_invisible(void *win_ptr, glctx_t *share_ctx,
96  int major_ver, int minor_ver, bool_t fwd_compat, bool_t debug);

98 /*
99  * Returns an opaque handle to the X-Plane main window that can be used
100  * in glctx_create_invisible to create a context with the same hardware
101  * target. This function must ONLY be called on the main X-Plane thread.
102  * The returned value is platform-specific as follows:
103  * 1) On Windows, this returns the HWND of the main X-Plane window.
104  * 2) On Linux, this returns the value of the "DISPLAY" environment
